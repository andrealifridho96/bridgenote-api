Get started

The Bridgenote API provides programmatic access to insert, retrieve, update and delete user data (id, status and position).

To use this API, you need an API token. Please contact us at andre.alifridho@gmail.com to get your own API token.

get user

To get user data you need to make a GET call to the following url :
http://localhost:8000/api/bridgenote/get
http://localhost:8000/api/bridgenote/get?id=


QUERY PARAMETERS

Name Type Field Description
X-Client-Token String Header Your API client token.
Id int Param (Optional) User Id to retrieve the data if you want to get the specific data by id

insert user

To insert user data you need to make a POST call to the following url :
http://localhost:8000/api/bridgenote/insert


QUERY PARAMETERS

Name Type Field Description
X-Client-Token String Header Your API client token.
status String form-data (Required) to insert status value "active" or "inactive"
position String form-data (Required) to insert position value

update user

To update user data you need to make a PUT call to the following url :
http://localhost:8000/api/bridgenote/update?id={id}&status={value}&position={value}
If you want to update just only one field just send one parameter (status or position).


QUERY PARAMETERS

Name Type Field Description
X-Client-Token String Header Your API client token.
id int param (Required) to get user you want to update
status String param (Required) to update status value
position String param (Required) to update position value

delete user

To delete user data you need to make a DELETE call to the following url :
http://localhost:8000/api/bridgenote/delete?id={id}


QUERY PARAMETERS

Name Type Field Description
X-Client-Token String Header Your API client token.
id int param (Required) to get user you want to update

get user social media

To get user data from social media you need to make a POST call to the following url :
http://localhost:8000/auth/facebook
http://localhost:8000/auth/google

Errors

The Bridgenote API uses the following error codes:

Error Code Meaning
401 Unknown or unvalid X-CLient-Token. This error appears if you use an unknown API token.
500 Internal server error
